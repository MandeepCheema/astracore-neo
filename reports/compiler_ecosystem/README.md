# reports/compiler_ecosystem/ — MLIR / IREE / TVM conformance evidence

Generated by `astracore conformance` (CLI) or directly via
`python -m tools.frontends.iree_conformance`. Addresses Track 3 item 9
of `docs/external_validation_options.md` — "TVM BYOC backend" — and its
companion "MLIR / IREE conformance" entry.

## What the baseline measures

The harness builds 15 canonical ONNX graphs (one per op class that
AstraCore's compiler lowers) with `onnx.helper`, then pushes each graph
through the three compiler-ecosystem front-end adapters we maintain:

| Adapter          | Intended source                                           |
|------------------|-----------------------------------------------------------|
| `mlir-stablehlo` | torch-mlir / IREE's `iree-import-onnx` / StableHLO export |
| `tvm-relay`      | TVM's `relay.frontend.from_onnx`                          |
| `jax-xla`        | JAX / XLA → ONNX round-trip                               |

Each adapter normalises on ONNX bytes → `NnGraph`. A case passes iff
its expected `OP_*` constants land in the resulting graph.

**First baseline: 45/45 pass** across 15 canonical ops × 3 adapters.
See `conformance.json` for machine-readable results + the full TVM BYOC
pattern table (20 entries), `conformance.md` for the per-case grid.

## TVM BYOC scaffold

`tools/frontends/tvm_byoc.py` is import-safe without the `tvm`
package — this lets the pattern table, target name, and JSON
introspection helper run in every CI environment. The TVM-gated entry
points (`register_astracore_target`, `partition_graph_for_astracore`,
`astracore_codegen`) raise a clear `ImportError` when `tvm` is missing
and execute the standard BYOC transform sequence
(`MergeComposite → AnnotateTarget → MergeCompilerRegions →
PartitionGraph`) when it is present.

End-to-end Relay partitioning is exercised by
`tests/test_compiler_ecosystem.py::test_tvm_byoc_partition_smoke_when_tvm_available`,
marked `@pytest.mark.integration` so it only runs where `tvm` is
installed.

## What this does NOT yet cover

- No real codegen → AstraCore binary. The `astracore_codegen` stub
  round-trips via ONNX → NnGraph today; the TIR → runtime-call lowering
  lands once the C-ABI does (same track as the F1 host driver).
- No IREE end-to-end compilation. We validate ONNX-as-StableHLO-proxy
  today; the full `iree-compile --iree-hal-target-backends=astracore`
  path lands with the C-ABI too.

## Reproduce

```bash
# Via CLI (writes here)
astracore conformance --out reports/compiler_ecosystem

# Restrict to a single adapter
astracore conformance --adapter mlir-stablehlo --out /tmp/probe

# Direct module invocation (same output)
python -m tools.frontends.iree_conformance --out reports/compiler_ecosystem
```
